# API-сервис сокращения ссылок

## 📌 Описание проекта

Необходимо разработать API-сервис для сокращения ссылок.  
Сервис позволяет:

- создавать короткие ссылки из длинных URL,
- перенаправлять пользователей по короткому коду,
- управлять созданными ссылками,
- получать статистику использования.

Примеры аналогичных сервисов:
- TinyURL
- Bitly

---

## ⚙️ Как это работает

1. Пользователь отправляет запрос `POST /links/shorten` с длинным URL.
2. Сервис генерирует уникальный короткий код.
3. Короткая ссылка возвращается пользователю.
4. При переходе по короткой ссылке (`GET /{short_code}`) происходит редирект на оригинальный URL.

---

# 🔴 Обязательный функционал

## 1. Управление короткими ссылками

### ➤ Создание короткой ссылки  
`POST /links/shorten`

Создает короткую ссылку на основе переданного оригинального URL.

Поддерживает:
- `custom_alias` — пользовательский короткий код (проверяется на уникальность).
- `expires_at` — время жизни ссылки (формат даты с точностью до минуты).

---

### ➤ Редирект  
`GET /links/{short_code}`

Перенаправляет пользователя на оригинальный URL.

---

### ➤ Удаление ссылки  
`DELETE /links/{short_code}`

Удаляет связь короткого кода с оригинальным URL.  
Доступно только зарегистрированным пользователям.

---

### ➤ Обновление ссылки  
`PUT /links/{short_code}`

Позволяет:
- изменить оригинальный URL  
или  
- привязать новую длинную ссылку к существующему short_code.

Доступно только зарегистрированным пользователям.

---

## 2. Статистика по ссылке

`GET /links/{short_code}/stats`

Возвращает:
- оригинальный URL
- дату создания
- количество переходов
- дату последнего использования

---

## 3. Поиск ссылки

`GET /links/search?original_url={url}`

Позволяет найти короткую ссылку по оригинальному URL.

---

## 4. Время жизни ссылки

При создании ссылки можно указать:
expires_at = YYYY-MM-DD HH:MM


После наступления указанного времени:
- ссылка автоматически удаляется
или
- становится недоступной.

# 🟡 Дополнительные функции (минимум 2)

Примеры возможной реализации:

### 1. Автоматическое удаление неиспользуемых ссылок
Удаление ссылок спустя N дней после последнего перехода.  
Значение N задается глобально.

### 2. История истекших ссылок
Отдельный endpoint для просмотра удаленных/истекших ссылок.

### 3. Группировка ссылок по проектам
Пользователь может объединять ссылки в проекты.

### 4. Создание ссылок без регистрации
Незарегистрированные пользователи могут создавать ссылки, но не управлять ими.

# 👤 Регистрация пользователей

Реализовать простую регистрацию и аутентификацию.

Требования:

- Изменение и удаление ссылок доступны только зарегистрированным пользователям.
- Создание и редирект доступны всем.
- В БД хранится информация о владельце ссылки (или факт отсутствия регистрации).

---

# 🗄 База данных и кэширование

## Основное хранилище
Рекомендуется использовать:
- PostgreSQL

Причины:
- поддержка индексов,
- удобный поиск,
- транзакционность.

---

## Кэширование

Обязательно использовать:
- Redis (или аналог)

Рекомендуется кэшировать:
- самые популярные ссылки,
- данные для редиректов,
- статистику.

При обновлении или удалении ссылки необходимо:
- очищать соответствующие записи в кэше.
